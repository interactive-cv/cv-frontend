# cv-frontend

Фронтенд для **персонального интерактивного сайта-резюме** — open-source-шаблон на Next.js 16 (App Router, TypeScript, Tailwind CSS). Обращается к собственному FastAPI-бэкенду ([cv-backend](https://github.com/interactive-cv/cv-backend)). Любой может развернуть такой сайт под своё CV: меняется только контент и домен.

Живой пример сервиса, собранного из этого кода: **[cv.libera.pro](https://cv.libera.pro)**.

Сайт задуман не как статичная «визитка», а как интерактивная витрина: кроме классического резюме здесь — кликабельный таймлайн проектов, граф связей технологий и **плавающий AI-чат**, в который рекрутёр может задать вопрос и получить ответ по фактам из CV.

## Технологии

- **Next.js 16 (App Router) + React 19** — серверный рендеринг (SSR) лендинга и страниц резюме
- **TypeScript (strict)** — строгая типизация, без `any`
- **Tailwind CSS v4 + @tailwindcss/typography** — оформление, тёмная/светлая тема (next-themes)
- **framer-motion** — анимации появления и переходов
- **react-force-graph-2d** — интерактивный граф знаний
- **react-markdown** — рендер ответов AI и страниц CV
- **AI/LLM-интеграция** — стриминг ответов модели (Server-Sent Events), плавающий чат-ассистент по фактам из CV, UI для AI-генерации откликов из вакансии
- **Jest (next/jest) + Testing Library** — 5 тестов на критичные компоненты

## Что внутри

| Страница / компонент | Что делает |
|----------------------|-----------|
| `/` лендинг | Hero с иконкой-логотипом, блок навыков, кликабельный таймлайн проектов с модальными окнами, граф знаний |
| `/{slug}` страница CV | SSR-рендер варианта резюме под конкретную вакансию (markdown → красивый текст, иконка-логотип) |
| `/{CODE}` короткая ссылка | Proxy перехватывает, резолв + трекинг session_id → редирект на нужный вариант CV |
| `/admin` админ-панель | Token-вход, список откликов с фильтром (вакансии/фриланс/конкурсы), inline-аналитика, tooltip уникальных посетителей, виджет «Ближайшие собеседования» |
| `/admin/new` AI-генерация | Форма (vacancy/freelance/contest) + выбор проектов + ТЗ из PDF → LLM генерирует CV и cover letter → split-редактор → публикация |
| `/admin/{id}` детальная | Табы: CV, отклик, оценка, заказ/вакансия, **собеседования** (CRUD + заметки до/после), **артефакты** (загрузка APK/видео, публичные ссылки `/dl/{code}`), детали (бюджет/рейтинг/ссылки + snapshot промпта), аналитика + PDF-скачивание |
| `/admin/settings` настройки | Мастер-CV (с AI-правкой), README, 7 промптов — редактируемые через админку |
| `/admin/chats` чаты HR | Список сессий с именами посетителей («Крепкий Кабан»), группировка по откликам, просмотр диалогов |
| Чат-виджет | Плавающая кнопка, потоковый ответ AI, typing-индикатор, markdown, **продолжение диалога** (cookie session_id, восстановление истории) |
| Тумблер темы | 🌙/☀ переключатель в шапке (next-themes), тёмная по умолчанию, выбор сохраняется в localStorage, тёплая «монашеская» светлая палитра |

## Интеграция AI (со стороны фронта)

AI-чат — ключевой элемент, который обращает на себя внимание рекрутёра. Реализация на клиенте:

- **Потоковая передача** ответа через `ReadableStream` — токены появляются по мере генерации, как в ChatGPT, а не после полной загрузки. Плюс анимированный typing-индикатор (три точки), пока модель «думает».
- **Защита от галлюцинаций лежит на бэкенде** (жёсткий системный промпт + e2e-тесты RAG) — фронт честно показывает то, что приходит. Запасной сценарий graceful degradation тоже на сервере: при сбое LLM приходит сообщение с контактами, виджет не падает.
- Сообщения пользователя рендерятся как plain text (без интерпретации ввода как разметки — безопасность), ответы AI — через `react-markdown`.

## SEO и шеринг

Ссылки на CV обычно кидают в мессенджеры, поэтому превью ссылки — первое впечатление:

- **Динамическая OG-картинка** (`opengraph-image.tsx`, генерируется через `ImageResponse`) — имя, стек, бейджи навыков на градиентном фоне
- **Персонализированные метаданные** для каждой страницы CV через `generateMetadata` — превью показывает название конкретного варианта резюме
- **favicon** — по умолчанию лежит `app/icon.png` (синяя иконка с «CV» + AI-бабблом, отражающая суть проекта). Чтобы заменить на свою: положите квадратный PNG (рекомендуется 512×512) в `app/icon.png`, либо используйте пример-генератор из `examples/icon-default.tsx` (скопируйте в `app/icon.tsx`). `robots.txt` и `sitemap.xml` генерируются Next.js автоматически.

## Развёртывание

Прод-окружение — собственный VPS с Docker Compose, фронт собирается multi-stage сборкой (`Dockerfile`): на этапе build ставятся зависимости и собирается production-билд Next.js (`output: "standalone"`), в runtime крутится минимальный образ без полного `node_modules`. Главная страница помечена `force-dynamic` — данные берутся из API при каждом запросе, поэтому сборка не зависит от доступности бэкенда.

Весь стек (вместе с бэкендом и nginx) поднимается одной командой из репозитория [cv-backend](https://github.com/interactive-cv/cv-backend) — `bash deploy.sh`.
